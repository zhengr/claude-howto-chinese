# 自我评估与学习路径顾问

全面的交互式评估，涵盖 Claude Code 在 10 个功能领域的熟练度，识别具体的技能差距，并生成个性化的学习路径以提升水平。

## 使用说明

### 第 1 步：欢迎语与选择评估深度

向用户展示评估深度选择：

使用 AskUserQuestion，包含以下选项：
- **快速评估** — "8 道题，约 2 分钟。确定你的总体水平（初级/中级/高级），并提供学习路径。"
- **深度评估** — "5 个类别的详细问题，约 5 分钟。提供每个主题的技能分数，识别具体差距，并构建优先级学习路径。"

如果用户选择 **快速评估**，转到第 2A 步。
如果用户选择 **深度评估**，转到第 2B 步。

---

### 第 2A 步：快速评估

展示两道多选题（AskUserQuestion 最多支持 4 个选项）：

**问题 1**（标题："基础"）：
"第 1/2 部分：以下 Claude Code 技能你已经掌握了哪些？"
选项：
1. "启动 Claude Code 并对话" — 我可以运行 `claude` 并与之交互
2. "创建/编辑过 CLAUDE.md" — 我已设置项目或个人记忆
3. "使用过 3+ 个斜杠命令" — 例如 /help、/compact、/model、/clear
4. "创建过自定义命令/技能" — 编写过 SKILL.md 或自定义命令文件

**问题 2**（标题："高级"）：
"第 2/2 部分：以下高级技能你已经掌握了哪些？"
选项：
1. "配置过 MCP 服务器" — 例如 GitHub、数据库或其他外部数据源
2. "设置过钩子" — 在 ~/.claude/settings.json 中配置过钩子
3. "创建/使用过子智能体" — 使用 .claude/agents/ 进行任务委派
4. "使用过打印模式（claude -p）" — 在非交互或 CI/CD 场景中使用过 `claude -p`

**评分：**
- 总计 0-2 项 = 第 1 级：初级
- 总计 3-5 项 = 第 2 级：中级
- 总计 6-8 项 = 第 3 级：高级

转到第 3 步，列出未勾选的具体项目作为差距。

---

### 第 2B 步：深度评估

展示 5 轮问题，每轮调用一次 AskUserQuestion。每轮涵盖 2 个相关功能领域。所有轮次均使用多选。

**重要提示**：AskUserQuestion 每题最多 4 个选项。每轮恰好 1 道题，4 个选项，涵盖 2 个主题（每个主题 2 个选项）。

---

**第 1 轮 — 斜杠命令与记忆**（标题："命令"）

"以下你做过哪些？选择所有适用的选项。"
选项：
1. "创建过自定义斜杠命令或技能" — 编写过带 frontmatter 的 SKILL.md 文件，或创建过 .claude/commands/ 文件
2. "在命令中使用过动态上下文" — 使用过 `$ARGUMENTS`、`$0`/`$1`、反引号 `!命令` 语法或 `@文件` 引用
3. "设置过项目 + 个人记忆" — 同时创建过项目 CLAUDE.md 和个人 ~/.claude/CLAUDE.md
4. "使用过记忆层级功能" — 理解 7 级优先级顺序、使用过 .claude/rules/ 目录、路径特定规则或 @import 语法

**第 1 轮评分：**
- 选项 1-2 映射到 **斜杠命令**（0-2 分）
- 选项 3-4 映射到 **记忆**（0-2 分）

---

**第 2 轮 — 技能与钩子**（标题："自动化"）

"以下你做过哪些？选择所有适用的选项。"
选项：
1. "安装并使用过自动触发技能" — 根据描述自动触发而无需手动调用的技能
2. "控制过技能触发行为" — 在 SKILL.md frontmatter 中使用过 `disable-model-invocation`、`user-invocable` 或 `context: fork` 及 agent 字段
3. "设置过 PreToolUse 或 PostToolUse 钩子" — 配置过在工具执行前/后运行的钩子（如命令验证器、自动格式化器）
4. "使用过钩子高级功能" — 配置过 prompt 类型钩子、SKILL.md 中的组件范围钩子、HTTP 钩子或带自定义 JSON 输出的钩子（updatedInput、systemMessage）

**第 2 轮评分：**
- 选项 1-2 映射到 **技能**（0-2 分）
- 选项 3-4 映射到 **钩子**（0-2 分）

---

**第 3 轮 — MCP 与子智能体**（标题："集成"）

"以下你做过哪些？选择所有适用的选项。"
选项：
1. "连接过 MCP 服务器并使用其工具" — 例如 GitHub MCP 获取 PR/issue、数据库 MCP 查询或其他外部数据源
2. "使用过 MCP 高级功能" — 项目级 .mcp.json、OAuth 认证、带 @提及的 MCP 资源、Tool Search 或 `claude mcp serve`
3. "创建或配置过自定义子智能体" — 在 .claude/agents/ 中定义过带自定义工具、模型或权限的智能体
4. "使用过子智能体高级功能" — 工作树隔离（`isolation: worktree`）、持久化智能体记忆、后台任务（Ctrl+B）、智能体白名单（`Task(agent_name)`）或智能体团队

**第 3 轮评分：**
- 选项 1-2 映射到 **MCP**（0-2 分）
- 选项 3-4 映射到 **子智能体**（0-2 分）

---

**第 4 轮 — 检查点与高级功能**（标题："高级用户"）

"以下你做过哪些？选择所有适用的选项。"
选项：
1. "使用过检查点进行安全实验" — 创建过检查点、使用过 Esc+Esc 或 /rewind、恢复过代码和/或对话，或使用过摘要选项
2. "使用过规划模式或扩展思考" — 通过 /plan、Shift+Tab 或 --permission-mode plan 激活规划；使用 Alt+T/Option+T 切换扩展思考
3. "配置过权限模式" — 使用过 acceptEdits、plan、dontAsk 或 bypassPermissions 模式
4. "使用过远程/桌面/Web 功能" — 使用过 `claude remote-control`、`claude --remote`、`/teleport`、`/desktop` 或 `claude -w` 工作树

**第 4 轮评分：**
- 选项 1 映射到 **检查点**（0-1 分）
- 选项 2-4 映射到 **高级功能**（0-3 分，上限 2 分）

---

**第 5 轮 — 插件与 CLI**（标题："精通"）

"以下你做过哪些？选择所有适用的选项。"
选项：
1. "安装或创建过插件" — 使用过市场中的捆绑插件，或创建过带 plugin.json 清单的 .claude-plugin/ 目录
2. "使用过插件高级功能" — 插件钩子、插件 MCP 服务器、LSP 配置、插件命名空间命令或 --plugin-dir 测试标志
3. "在脚本或 CI/CD 中使用过打印模式" — 使用过 --output-format json、--max-turns、管道输入或集成到 GitHub Actions/CI 流水线
4. "使用过 CLI 高级功能" — 会话恢复（-c/-r）、--agents 标志、--json-schema 结构化输出、--fallback-model、--from-pr 或批处理循环

**第 5 轮评分：**
- 选项 1-2 映射到 **插件**（0-2 分）
- 选项 3-4 映射到 **CLI**（0-2 分）

### 第 4 步：生成个性化学习路径

根据评估结果，生成针对用户差距的学习路径。不要简单重复通用等级路径 — 需要适配。

#### 路径生成规则

1. **跳过已掌握主题**：如果某主题得分为 2/2，则不包含在路径中。
2. **按依赖关系排序**：斜杠命令在技能之前，记忆在子智能体之前等。依赖关系为：
   - 斜杠命令（无依赖） -> 技能（依赖斜杠命令）
   - 记忆（无依赖） -> 子智能体（依赖记忆）
   - CLI 基础（无依赖） -> CLI 精通（依赖所有）
   - 检查点（无依赖）
   - 钩子（依赖斜杠命令）
   - MCP（无依赖） -> 插件（依赖 MCP、技能、钩子）
   - 高级功能（依赖所有前述内容）
3. **对于 1/2 分的主题**：推荐"深入探索" — 链接到其缺少的特定高级部分。
4. **估算时间**：仅汇总需要学习/复习的主题。
5. **分组为阶段**：将剩余主题按逻辑组织为每阶段 2-3 个主题。

#### 路径输出格式

```markdown
### 你的个性化学习路径

**估算时间**：~N 小时（根据你的现有技能调整）

#### 阶段 1：[阶段名称]（~N 小时）
[仅在他们有差距的领域包含]

**[主题名称]** — [从零学习 / 深入探索高级功能]
- 教程：[教程目录链接]
- 重点关注：[他们需要学习的特定概念]
- 核心练习：[一个具体练习]
- 成功标志：[具体的成功标准]

**[主题名称]** — ...

---

#### 阶段 2：[阶段名称]（~N 小时）
...

---

### 推荐实践项目

根据你的差距，尝试以下实战练习来巩固学习成果：

1. **[项目名称]**：[结合 2-3 个差距领域的 1 行描述]
2. **[项目名称]**：[1 行描述]
3. **[项目名称]**：[1 行描述]
```

#### 主题特定推荐

当某主题存在差距时，使用以下具体建议：

**斜杠命令（0 分）**：
- 教程：[01-slash-commands/](../../../01-slash-commands/)
- 重点关注：内置命令参考、创建第一个 SKILL.md、`$ARGUMENTS` 语法
- 核心练习：创建 `/optimize` 命令并测试
- 成功标志：能创建带参数和动态上下文的自定义技能

**斜杠命令（1 分 — 复习）**：
- 重点关注：`!反引号` 语法的动态上下文、`@文件` 引用、`disable-model-invocation` 与 `user-invocable` 控制
- 成功标志：能创建注入实时命令输出并控制自身触发行为的技能

**记忆（0 分）**：
- 教程：[02-memory/](../../../02-memory/)
- 重点关注：CLAUDE.md 创建、`/init` 和 `/memory` 命令、`#` 前缀快速更新
- 核心练习：创建包含编码标准的项目 CLAUDE.md
- 成功标志：Claude 跨会话记住你的偏好

**记忆（1 分 — 复习）**：
- 重点关注：7 级层级和优先级顺序、.claude/rules/ 目录的路径特定规则、`@import` 语法（最大深度 5）、自动记忆 MEMORY.md（200 行限制）
- 成功标志：拥有不同目录的模块化规则并理解完整层级

**技能（0 分）**：
- 教程：[03-skills/](../../../03-skills/)
- 重点关注：SKILL.md 格式、通过 description 字段自动触发、渐进式披露（3 级加载）
- 核心练习：安装 code-review 技能并验证其自动触发
- 成功标志：技能根据对话上下文自动激活

**技能（1 分 — 复习）**：
- 重点关注：`context: fork` 与 `agent` 字段用于子智能体执行、`disable-model-invocation` 与 `user-invocable`、2% 上下文预算、捆绑资源（scripts/、references/、assets/）
- 成功标志：能创建在子智能体中运行并带分叉上下文的技能

**钩子（0 分）**：
- 教程：[06-hooks/](../../../06-hooks/)
- 重点关注：配置结构（matcher + hooks 数组）、PreToolUse/PostToolUse 事件、退出码（0=成功，2=阻断）、JSON 输入/输出格式
- 核心练习：创建验证 Bash 命令的 PreToolUse 钩子
- 成功标志：钩子在执行前阻断危险命令

**钩子（1 分 — 复习）**：
- 重点关注：全部 25 个钩子事件（含 PostToolUseFailure、StopFailure、TaskCreated、CwdChanged、FileChanged、PostCompact、Elicitation、ElicitationResult）、4 种钩子类型（command、http、prompt、agent）、SKILL.md frontmatter 中的组件范围钩子、带 allowedEnvVars 的 HTTP 钩子、SessionStart/CwdChanged/FileChanged 的 `CLAUDE_ENV_FILE`
- 成功标志：能创建基于提示的 Stop 钩子和技能中的组件范围钩子

**MCP（0 分）**：
- 教程：[05-mcp/](../../../05-mcp/)
- 重点关注：`claude mcp add` 命令、传输类型（推荐 HTTP）、GitHub MCP 设置、环境变量展开
- 核心练习：添加 GitHub MCP 服务器并查询 PR
- 成功标志：能通过 MCP 从外部服务查询实时数据

**MCP（1 分 — 复习）**：
- 重点关注：项目级 .mcp.json（需要团队审批）、OAuth 2.0 认证、带 `@server:resource` 提及的 MCP 资源、Tool Search（ENABLE_TOOL_SEARCH）、`claude mcp serve`、输出限制（10k/25k/50k）
- 成功标志：拥有项目 .mcp.json 并理解 Tool Search 自动模式

**子智能体（0 分）**：
- 教程：[04-subagents/](../../../04-subagents/)
- 重点关注：智能体文件格式（.claude/agents/*.md）、内置智能体（general-purpose、Plan、Explore）、tools/model/permissionMode 配置
- 核心练习：创建 code-reviewer 子智能体并测试委派
- 成功标志：Claude 将代码审查委派给你的自定义智能体

**子智能体（1 分 — 复习）**：
- 重点关注：工作树隔离（`isolation: worktree`）、持久化智能体记忆（`memory` 字段带作用域）、后台智能体（Ctrl+B/Ctrl+F）、智能体白名单（`Task(agent_name)`）、智能体团队（`--teammate-mode`）
- 成功标志：拥有带持久记忆和工作树隔离的子智能体

**检查点（0 分）**：
- 教程：[08-checkpoints/](../../../08-checkpoints/)
- 重点关注：Esc+Esc 和 /rewind 访问、5 种回退选项（恢复代码+对话、恢复对话、恢复代码、摘要、取消）、限制（bash 文件系统操作不追踪）
- 核心练习：做实验性修改，然后回退恢复
- 成功标志：能自信地实验，知道可以回退

**高级功能（0 分）**：
- 教程：[09-advanced-features/](../../../09-advanced-features/)
- 重点关注：规划模式（/plan 或 Shift+Tab）、权限模式（5 种类型）、扩展思考（Alt+T 切换）
- 核心练习：使用规划模式设计功能，然后实现它
- 成功标志：能熟练切换规划和实现模式

**高级功能（1 分 — 复习）**：
- 重点关注：远程控制（`claude remote-control`）、Web 会话（`claude --remote`）、桌面交接（`/desktop`）、工作树（`claude -w`）、任务列表（Ctrl+T）、企业托管设置
- 成功标志：能在 CLI、Web 和桌面之间交接会话

**插件（0 分）**：
- 教程：[07-plugins/](../../../07-plugins/)
- 重点关注：插件结构（.claude-plugin/plugin.json）、插件捆绑内容（命令、智能体、MCP、钩子、设置）、从市场安装
- 核心练习：安装插件并探索其组件
- 成功标志：理解插件何时优于独立组件

**插件（1 分 — 复习）**：
- 重点关注：创建 plugin.json 清单、插件钩子（hooks/hooks.json）、LSP 配置（.lsp.json）、`${CLAUDE_PLUGIN_ROOT}` 变量、--plugin-dir 测试、市场发布
- 成功标志：能为团队创建并测试插件

## 错误处理

### 用户在某轮未选择任何项
视为该轮主题 0 分。继续下一轮。

### 用户所有轮次均未选择
分配第 1 级：初级。鼓励从头开始。输出完整的第 1 级学习路径。

### 用户想重新评估
从第 1 步重新开始，进行全新评估。

### 用户不同意其等级
承认其偏好。询问其认同的等级。展示其选择等级的学习路径，并检查可能遗漏的先决条件。

### 用户询问特定主题
如果在评估过程中用户说"告诉我关于钩子"或"我想学习 MCP"之类的话，记录它。呈现结果时，无论分数如何，在学习路径中突出该主题。
