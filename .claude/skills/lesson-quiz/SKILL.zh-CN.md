# 课程测验

互动式测验，通过 8-10 道题目检验对特定 Claude Code 课程的理解，提供逐题反馈，并标识出需要复习的领域。

## 使用说明

### 第 1 步：确定课程

如果用户提供了课程参数（如 `/lesson-quiz hooks` 或 `/lesson-quiz 03`），将其映射到对应的课程目录：

**课程映射表：**
- `01`, `slash-commands`, `commands` → 01-slash-commands
- `02`, `memory` → 02-memory
- `03`, `skills` → 03-skills
- `04`, `subagents`, `agents` → 04-subagents
- `05`, `mcp` → 05-mcp
- `06`, `hooks` → 06-hooks
- `07`, `plugins` → 07-plugins
- `08`, `checkpoints`, `checkpoint` → 08-checkpoints
- `09`, `advanced`, `advanced-features` → 09-advanced-features
- `10`, `cli` → 10-cli

如果未提供参数，使用 AskUserQuestion 展示选择菜单：

**问题 1**（标题："课程"）：
"你想对哪节课进行测验？"
选项：
1. "斜杠命令（01）" — 自定义命令、技能、前置元数据、参数
2. "记忆（02）" — CLAUDE.md、记忆层级、规则、自动记忆
3. "技能 Skills（03）" — 渐进式信息展示、自动调用、SKILL.md
4. "子代理（04）" — 任务委派、代理配置、隔离

**问题 2**（标题："课程"）：
"你想对哪节课进行测验？（续）"
选项：
1. "MCP（05）" — 外部集成、传输、服务器、工具搜索
2. "钩子（06）" — 事件自动化、PreToolUse、退出码、JSON I/O
3. "插件 Plugins（07）" — 打包方案、市场、plugin.json
4. "更多课程..." — 检查点、高级功能、命令行界面

如果选择了"更多课程..."，展示：

**问题 3**（标题："课程"）：
"选择你的课程："
选项：
1. "检查点（08）" — 回退、恢复、安全实验
2. "高级功能（09）" — 规划、权限、打印模式、深度思考
3. "命令行界面参考（10）" — 命令行选项、输出格式、脚本、管道

### 第 2 步：阅读课程内容

阅读课程 README.md 文件以刷新上下文：
- 读取文件：`<lesson-directory>/README.md`

然后从 `references/question-bank.md` 中获取该课程的题库。题库提供每节课 10 道预写题目，包含正确答案和解释。

### 第 3 步：呈现测验

询问用户测验的时机上下文：

使用 AskUserQuestion（标题："时机"）：
"你是在课程的哪个阶段进行此测验？"
选项：
1. "课前（预测试）" — 我还未阅读课程内容，测试已有知识
2. "课中（进度检查）" — 我正在学习过程中
3. "课后（掌握度检验）" — 我已学完课程，想验证理解程度

这个上下文影响结果的呈现方式（见第 5 步）。

### 第 4 步：分轮呈现题目

从题库中取 10 道题目，每轮 2 道（共 5 轮）。每道题目使用 AskUserQuestion 呈现，附带 3-4 个选项。

**重要提示**：使用 AskUserQuestion，每题最多 4 个选项，每轮 2 道题目。

每轮展示 2 道题目。全部 5 轮完成后，进入评分环节。

**每轮题目格式：**

题库中每道题目包含：
- `question`：题目文本
- `options`：3-4 个选项（其中一个是正确答案）
- `correct`：正确答案标识
- `explanation`：答案解释
- `category`："conceptual"（概念型）或 "practical"（实操型）

使用 AskUserQuestion 呈现每道题目，记录用户的答案。

### 第 5 步：评分并呈现结果

所有轮次完成后，计算分数并呈现结果。

**评分规则：**
- 每题答对 = 1 分
- 满分 = 10 分

**等级标准：**
- 9-10：掌握（Mastered）— 理解优秀
- 7-8：熟练（Proficient）— 基础扎实，存在细微小盲区
- 5-6：发展中（Developing）— 基础概念已掌握，需复习
- 3-4：初级（Beginning）— 存在明显知识缺口，建议复习
- 0-2：尚未起步（Not yet）— 建议从头学习本课程

**输出格式：**

```markdown
## 课程测验结果：[课程名称]

**得分：N/10** — [等级标识]
**测验时机**：[课前 / 课中 / 课后]完成测验
**题目分类**：概念型答对 N 题，实操型答对 N 题

### 逐题结果

| # | 类型 | 题目（简述） | 你的答案 | 结果 |
|---|----------|-----------------|-------------|--------|
| 1 | 概念型 | [题目摘要] | [所选答案] | [正确 / 错误] |
| 2 | 实操型 | ... | ... | ... |
| ... | ... | ... | ... | ... |

### 错题 — 请复习

[对每道错题，展示：]

**Q[N]: [完整题目]**
- 你的答案：[所选内容]
- 正确答案：[正确选项]
- 解释：[为何正确]
- 复习建议：[对应课程 README 中应重读的具体章节]

### [时机相关提示]

[如果是课前]：
**预测验得分：N/10。** 这为你建立了一个学习基线！集中精力学习错题涉及的知识点。完成课程后，重新参加测验以评估进步。

[如果是课中]：
**进度检查：N/10。** [7 分以上：进度喜人，继续前进！4-6 分：在继续学习前复习一下错题主题。4 分以下：建议从头重读课程。]

[如果是课后]：
**掌握度检验：N/10。** [9-10 分：你已完全掌握课程内容，可继续下一课。7-8 分：还差一点——复习错过的知识点后重测。7 分以下：花时间多学习课程内容，尤其关注上面标注的章节。]

### 建议的下一步

[根据得分和时间：]
- [已掌握]：继续学习下一课：[下一课链接]
- [熟练]：复习这些具体章节，然后重测：[章节列表]
- [发展中或以下]：重读完整课程：[课程链接]。重点学习：[薄弱环节列表]
- [提示]："你想重测本节课、尝试不同课程，还是就特定知识点获取帮助？"
```

### 第 6 步：提供后续选项

展示结果后，使用 AskUserQuestion：

"你接下来想做什么？"
选项：
1. "重测本节课" — 再次尝试同一课程的测验
2. "测验另一节课" — 切换到不同的课程
3. "解释我遗漏的知识点" — 获取错题的详细解释
4. "结束" — 结束本次测验

如果选择**重测**：返回第 4 步（跳过时机问题，沿用相同时机）。
如果选择**测验另一节课**：返回第 1 步。
如果选择**解释知识点**：询问题目编号，然后从课程 README.md 中读取相关章节并结合示例进行解释。

## 异常处理

### 无效的课程参数
如果参数不匹配任何课程，展示有效课程列表并请用户选择。

### 测验中途退出
如果用户在任何轮次中表示想停止，按已答题题目展示部分结果。

### 课程 README 未找到
如果 README.md 文件在预期路径不存在，通知用户并建议检查项目结构。

## 验证

### 触发测试用例

**应触发：**
- "测测我对 hooks 的了解"
- "课程测验"
- "检验我对第 3 课的知识"
- "MCP 练习测验"
- "我理解不理解 skills"
- "考考我对斜杠命令的了解"
- "lesson-quiz 06"
- "考核我对 checkpoints 的掌握"
- "我对 CLI 了解多少"
- "在学习记忆课前帮我测试一下"

**不应触发：**
- "评估我的整体水平"（使用 /self-assessment）
- "帮我解释一下 hooks"
- "创建一个钩子"
- "什么是 MCP"
- "审查我的代码"
