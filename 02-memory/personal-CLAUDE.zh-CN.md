# 我的开发偏好

## 关于我
- **经验水平**：8 年全栈开发
- **首选语言**：TypeScript, Python
- **沟通风格**：直接，附带示例
- **学习风格**：可视化图解配合代码

## 代码偏好

### 错误处理
我倾向于使用 try-catch 块和有意义的错误消息进行显式错误处理。
避免使用泛泛的错误信息。始终记录错误以便调试。

### 注释
注释应用于解释"为什么"，而非"是什么"。代码应该是自解释的。
注释应说明业务逻辑或不明显的决策。

### 测试
我倾向于使用 TDD（测试驱动开发）。
先编写测试，再实现功能。
关注行为，而非实现细节。

### 架构
我倾向于模块化、松耦合设计。
使用依赖注入以提高可测试性。
分离关注点（控制器、服务、仓储）。

## 调试偏好
- 使用带前缀的 console.log：`[DEBUG]`
- 包含上下文：函数名、相关变量
- 尽可能使用堆栈跟踪
- 日志中始终包含时间戳

## 沟通
- 用图解解释复杂概念
- 在解释理论之前先展示具体例子
- 包含修改前后的代码片段
- 在最后总结要点

## 项目组织
我这样组织我的项目：
```
project/
  ├── src/
  │   ├── api/
  │   ├── services/
  │   ├── models/
  │   └── utils/
  ├── tests/
  ├── docs/
  └── docker/
```
