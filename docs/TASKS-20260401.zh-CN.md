### 支柱 4：新手入门

- [ ] **T-001** — 编写 `scripts/quickstart.sh`

  - 复制第一个斜杠命令 + CLAUDE.md + 技能到用户设置

  - 幂等：跳过已存在的文件并发出警告，绝不覆盖
  - 完成时打印后续步骤

- [ ] **T-002** — 创建 `QUICKSTART.md`（前 15 分钟可视化指南）
  - 注释终端步骤作为代码块（尽可能避免 PNG 截图；使用 ASCII 或 Mermaid）
  - 记录哪些命令产生每个视觉效果以便重新捕获

- [ ] **T-003** — 为所有 10 个模块 README 添加难度徽章
  - 格式：每个模块 README 顶部显示 `> 🟢 **Beginner** | ⏱ 30 minutes`
  - 在徽章下方添加"你将构建什么"要点预览

### 支柱 7：活跃课程

- [ ] **T-004** — 创建 `WHATS-NEW.md`
  - 格式：`## CC vX.X — DATE`，每个功能变更为要点列表
  - 至少为当前 CC 版本添加一个条目

- [ ] **T-005** — 为所有 10 个模块 README 添加版本徽章
  - 格式：`> ✅ Verified against Claude Code **vX.X.X** · Last verified: YYYY-MM-DD`
  - 为每个模块的 frontmatter 添加 `cc_version_verified`

- [ ] **T-006** — 创建 `.github/workflows/staleness-check.yml`
  - 调度：每周一 09:00 UTC
  - 从 frontmatter 读取每个模块的 `last_verified` 日期（使用 `yq` 或 10 行 Python 脚本）
  - 为任何 30+ 天未验证的模块创建 GitHub Issue
  - 如已存在相同标题的未关闭 Issue 则跳过

### 支柱 6：内容质量（CLAUDE.md）

### 支柱 2：AI 智能体索引（生成器）

  - 输出 `AGENT-INDEX.md`（人类可读摘要）
